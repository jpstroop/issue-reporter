#6184 Paper Prototype attaching a Transcript to a Video

Creator:
Trey Pendragon
Created:
2024-01-25 22:34:25
Status:
Open
Comments:
Oh, Hey! at 2024-01-31 22:16:25 :

Resource Show Page:

Notes:

  • Visibility will always be "Private" on a Resource if any FileSets contain video and the user has not yet indicated that a video does not have any audio that can be captioned (silent, only background music, etc.) or that the video has audio that can be captioned but a VTT file has not been uploaded yet in its original language (aka, "Visibility Criteria"). A warning/notice of this constraint should diplay to the user next to any visibility indicator.
  • Question: Do we set a default for "Has audio that can be captioned?" or is there a null value, meaning the user has not made any indication?

File Manager:

Notes:

  • Any video FileSets that do not meet the "Visibility Criteria" should show a warning to the user. The warning should provide a link to the screen where one can add captions (or whatever file is required) for that FileSet.

FileSet Show Page:

Notes:

  • Supplemental files (FileMetadata?) may be uploaded to the list of files.
  • If a file is deleted from this screen that affects the visibility of the resource (i.e. Captions in the original language), the user should be warned and confirm before deleting.

FileSet Edit Page:

Notes:

  • The only addition here is the "Has transcribable audio?" This indicates that there is no default and the user MUST make a selection before the resource can become visible.
  • If we require "Captions" (not just transcriptions) then we will need to change this language. Again, a "what's this" reveal would be helpful here.
  • If changing the value of that field will change the visibility of the resource, this should be indicated to the user before they click "Save".

Suplemental File Create/Upload Page:

Notes:

  • We will need to add these four options: Transcript, Subtitles, Closed Caption, Audio Description (See WCAG)
  • Each type can have many language files, but each can have only one file for any given language.
  • If the user selects a type and language combination that already has a file associated with it, notify the user that a file exists and will be overwritten by the new file.
  • There should be a "what's this?" reveal for users to ensure they understand the differences between types such as Transcripts, Captions, Subtitles, and Audio Descriptions

Note: Annotations of above images with descriptions of UI changes are in progress!
