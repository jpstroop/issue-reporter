#497 Map 2021 Senior Thesis items from certificate programs to their DataSpace collections.

Creator:
Kate Lynch
Created:
2021-10-07 19:56:40
Status:
Open
Comments:
Kate Lynch at 2021-10-15 19:16:16 :

As stated in the linked issue from 2020, the following action run at dspace CLI prompt resolves this issue:

cert_handle = '88435/dsp01c247ds15b'
cert_collection = DSpace::CLI::SeniorThesisCollection.find_for_handle(cert_handle)
query = DSpace::CLI::SeniorThesisCommunity.find_by_class_year('2021')
cert_program = 'Theater Program'
sub_query = query.find_by_certificate_program(cert_program)
unmapped = sub_query.results.select { |i| i.collections.select { |c| c.title.include?(cert_program) }.empty? }
unmapped.map { |i| cert_collection.add_item(i); cert_collection.update }
cert_collection.index

Where cert_handle is the certificate program ark to be mapped (consult the table above), the matching pu.certificate value from the item's metadata (for example Theater Program, Creative Writing Program), and the parameter in the call to find_by_class_year matching the current year.

I ran this code against the following item, after adjusting the class year to 2021, and adding a pu.certificate field instance with the value Theater Program: https://dataspace-dev.princeton.edu/handle/99999/fk4321d72q

I also adjusted the following item on staging (imported in 2020) to be in the class year 2021, with the pu.certificate value of Theater Program.: https://dataspace-staging.princeton.edu/handle/88435/dsp01xw42nb94p

We should be safe to run this on production ASAP.
