  • To remove the redundant first library label and keep only the location label that we retrieve from Alma. This means that in the orangelight search results page we will display "Architecture Library - Architecture New Book Shelf" vs "Architecture Library - Architecture Library - Architecture New Book Shelf".
  • In some of the $stacks locations the location label is the same as the library label. This is not consistent though. In this case I will remove the duplicate string/label for any $stacks locations that carry the same label as the library. For example in the case of "firestone$stacks", we will display in orangelight "Firestone Library" vs "Firestone Library - Firestone Library".

This solution will result in the same orangelight UI as what we have now in production.

Creator:

Status:

Comments:
Kevin McElwee at 2021-03-12 17:14:50 :

Maybe I should start making gists of these for safe keeping

SELECT (
    SELECT text_value as community_name
    FROM metadatavalue
    WHERE resource_type_id = 4 AND resource_id = community_id AND metadata_field_id = 64
),(
    SELECT text_value as collection_name
    FROM metadatavalue 
    WHERE resource_type_id = 3 AND resource_id = community2collection.collection_id AND metadata_field_id = 64
),handle,bitstream_id
FROM handle 
INNER JOIN item ON (item.item_id = handle.resource_id)
INNER JOIN item2bundle ON (item.item_id = item2bundle.item_id)
INNER JOIN bundle2bitstream ON (item2bundle.bundle_id = bundle2bitstream.bundle_id)
INNER JOIN collection2item on collection2item.item_id = item.item_id
INNER JOIN community2collection on community2collection.collection_id = collection2item.collection_id
WHERE bitstream_id IN (
    SELECT resource_id
    FROM metadatavalue WHERE resource_type_id = 0 
    GROUP BY resource_id 
    HAVING NOT (64 = ANY(ARRAY_AGG(metadata_field_id)))
);
