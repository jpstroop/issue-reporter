Comments:
hackartisan at 2021-03-29 16:00:16 :

Okay I traced this through and I think it makes more sense for it to be a string.

The CDL::EligibleItemService is used by a CDL::ChargeManager to manage CDL::ChargedItems in CDL::ResourceChargeLists. The item_id is ultimately stored as a string in the charged item: https://github.com/pulibrary/figgy/blob/0755b93ef23156b493e67281d97b163a925696ba/app/resources/cdl/charged_item.rb#L5

The number of available items is computed such that I would expect both to be strings: https://github.com/pulibrary/figgy/blob/0755b93ef23156b493e67281d97b163a925696ba/app/resources/cdl/charge_manager.rb#L122

And in fact the item ids used in the charge manager specs are all strings.
