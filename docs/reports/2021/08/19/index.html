#1198 Index temp locations

Creator:
hackartisan
Created:
2021-04-01 20:46:39
Status:
Open
Comments:
mzelesky at 2021-08-19 15:14:29 :

Possible approach:

In parsing a MARCXML record from the publishing job, evaluate all items under each holding (linked by the subfield 0 in the 876 and the subfield 8 in the 852). If the item's current location (876|y and 876|z) is the same as the holding location, store the item under that holding ID.

If the item's current location is different from the holding location (the item is in a temp location), create a 'fake' holding for that location and store the item under that fake holding. Any subsequent items in that current location found on the bib record that differ from their parent holding locations also get stored under that fake holding. The same process needs to happen for all location variances found.

This will also allow us to show the items under a holding, which is a feature from Voyager we had to abandon.
