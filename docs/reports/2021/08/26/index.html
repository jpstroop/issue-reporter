#2685 Display long-form reference values for numismatics

Creator:
Kate Lynch
Created:
2021-08-24 15:23:17
Status:
Open
Comments:
Christina Chortaria at 2021-08-26 13:19:43 :

I dont see this information in the orangelight document. Do we need to add more information in figgy?

Kate Lynch at 2021-08-26 13:53:09 :

@eliotjordan Would we just pull the full figgy_title value from the Reference in Figgy into Orangelight instead of the short value? For the example coin above, that would be "Price, Martin Jessop Price, The Coinage in the Name of Alexander the Great and Philip Arrhidaeus: A British Museum Catalogue, 1991" which doesn't match the example text that Alan provided, but I don't see another place where we could get a value like what Alan is suggesting. Is there one?

Eliot Jordan at 2021-08-26 17:24:04 :

The text for the reference is pulled from the citation decorator: https://github.com/pulibrary/figgy/blob/main/app/resources/numismatics/issues/numismatics/issue_decorator.rb#L77

You can make changes there: https://github.com/pulibrary/figgy/blob/main/app/resources/numismatics/citations/numismatics/citation_decorator.rb#L32

Use the full title instead of the short title: "#{decorated_numismatic_reference&.title} #{part} #{number}"

See also: https://github.com/pulibrary/figgy/blob/main/app/resources/numismatics/coins/orangelight_coin_builder.rb#L110

Also, at the moment, only issue references are included (that was the initial request from Alan when we started this work). We might want to add coin references as well in combined field. That might be a separate ticket, though.
