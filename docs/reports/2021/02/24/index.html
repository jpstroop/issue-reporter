#639 Requests for "/collections/eadid/componentid" should redirect to "/catalog/eadid_componentid"

Creator:
Trey Pendragon
Created:
2021-02-02 20:54:43
Status:
Open
Comments:
James R. Griffin III at 2021-02-24 14:56:47 :

For clarification, an eadid here refers to the finding aid identifier. In cases such as https://findingaids-beta.princeton.edu/catalog/C0101_c013317, C0101 is the finding aid identifier/EAD ID. These are migrated from the legacy platform, using the legacy URL structure: https://findingaids.princeton.edu/collections/C0101.

James R. Griffin III at 2021-02-24 15:07:03 :

Component IDs are individual levels of nested archival material items within a collection description. Component IDs are "levels" of archival collection description. For example:

  • Archives of Charles Scribner's Sons, 1786-2004, bulk 1880/1979
    • Series 1: Advertising Records, 1917-1964 (large gaps), 1917-1964
    • Subseries 1A: Advertising Records, 1917-1950

...where each component leads to a more nested level of description. As each collection description is a tree node, the leaf node for each is typically a level of description which indicates that a physical box or folder is being described.
