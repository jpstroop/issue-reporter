#1038 Document dumptypes and dumpfile types

Creator:
hackartisan
Created:
2021-02-02 21:16:55
Status:
Open
Comments:
hackartisan at 2021-02-04 14:44:18 :

dump_types

label: 'Changed Records'

constant: 'CHANGEDRECORDS' Bib records from incremental dump. This dump depends on the mergedids in order to work.

label: 'All Bib IDs'

constant: 'BIB_IDS' There are none of these currently in bibdata events. This is just a list of bib ids. Can't find evidence of use in the code.

label: 'All Bib IDs with Holdings'

constant: 'MERGED_IDS' Rake task finds all bib IDS and merges in the holdings. This is used to figure out which bibs we need for the changed records dump. See reference to earlier_merged_dump in Dump#diff_since-last

label: 'All Records'

constant: 'ALL_RECORDS' All bibs in a full dump

label: 'Updated Princeton ReCAP Records'

constant: 'PRINCETONRECAP' cron: https://github.com/pulibrary/marcliberation/blob/cd7226c1e3b97a630c5e334dc1004916e985d0ef/config/schedule.rb#L75 These are created to be sent to SCSB. We do not load these files locally with traject. These are currently broken and fixing them has been on hold.

label: 'Updated Partner ReCAP Records'

constant: 'PARTNERRECAP' We pull the latest updates from the partners (columbia, nypl). Then we run these through traject to index into our catalog. https://github.com/pulibrary/marcliberation/blob/cd7226c1e3b97a630c5e334dc1004916e985d0ef/config/schedule.rb#L80

dumpfiletypes

label: 'Updated Records'

constant: 'UPDATED_RECORDS'

label: 'New Records'

constant: 'NEW_RECORDS' Not used

label: 'All Bib IDs with Holdings'

constant: 'MERGED_IDS'

label: 'All Bib IDs'

constant: 'BIB_IDS'

label: 'All Bib Records'

constant: 'BIB_RECORDS'

label: 'Updated ReCAP Records'

constant: 'RECAP_RECORDS'

label: 'Log File'

constant: 'LOGFILE' The partnerrecap dump will have a log file along with the file of records
