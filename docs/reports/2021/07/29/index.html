#1518 Index all incremental dumps created since the most recent full dump

Creator:
hackartisan
Created:
2021-07-12 20:23:07
Status:
Open
Comments:
hackartisan at 2021-07-29 17:56:56 :

Here's a proposal for an IndexManager to coordinate indexing alma full and incremental dumps.

class IndexManager < ActiveRecord::Base
  solr_collection string
  dump_in_progress foreign key
  dumps_complete [foreign key]
  active boolean

  # returns the active IndexManager for the given collection
  def self.current(solr_collection)

  # finds and indexes the next incremental dump. no-op if there's a dump_in_progress.
  def next_incremental

Running a new full index on catalog-alma-rebuild: - Set current Index Manager for the rebuild collection to active:false - empty the rebuild index via the solr API - create a new IndexManager for the rebuild collection and set it to active - queue up the initial full dump - when a full dump finishes, it calls IndexManager.current(collection).next_incremental - the full dump index jobs will have to store a reference to the index manager or the collection they're going into for this to work. - sidekiq batches make an action like this possible via the batch callback

Ensuring all incrementals get indexed, and in the correct order - When a new incremental file gets transferred from sftp, call nextincremental on the configured primary SOLR_COLLECTION - When any incremental file finishes indexing, call nextincremental on the collection it indexed into - sidekiq batches make an action like this possible via the batch callback

Instead of configuring SOLR_URL and SOLR_REINDEX_URL in env, we configure SOLR_BASE_URL, SOLR_COLLECTION, and SOLR_REINDEX_COLLECTION.

Trey Pendragon at 2021-07-29 19:27:12 :

Proposal:

See sidekiq batch example: https://github.com/mperham/sidekiq/wiki/Really-Complex-Workflows-with-Batches

Create a "super-batch" for each indexing operation.

When a Dump completes, look for any dumps after that dump and add it to the batch.

If another super-batch tries to enqueue, just fail out if one exists already.
