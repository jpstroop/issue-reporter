#495 Debug Getty harvesting

Creator:
Kate Lynch
Created:
2021-12-16 14:22:01
Status:
Closed
Comments:
Kate Lynch at 2021-12-20 15:54:20 :

This appears to be a data issue. The harvester checks first for association with the Cicognara digital collection, then for the presence of a DCL identifier, and then checks for the presence of a IIIF manifest URL. Only if all three of these checks pass, is the record then ingested.

I determined the following by debugging the harvester, for items with the value "Getty Research Institute" in the Contributing Library field: * Passed Cicognara check: 1298 records * Passed DCL number check: 1272 records * Passed manifest URL check: 5

So it looks like there is a problem getting manifests into the correct place in the records. It looks like it's completely missing from the Getty records rather than just in the wrong place.

  • Example with Cicognara collection association, DCL number, and manifest URL - this one indexes correctly
  • Example with Cicognara collection association and DCL number but no manifest URL - this one is skipped but looks like it should be included.

We need to get the missing manifest URL values into the records from the Getty, or adjust the check. The items without manifest URLs are using the Archive.org BookReader.
