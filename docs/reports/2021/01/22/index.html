#922 Update contained_in_s field in traject for Alma

Creator:
hackartisan
Created:
2020-12-04 20:20:49
Status:
Open
Comments:
mzelesky at 2021-01-22 15:29:33 :

This is a working model for how to retrieve availability and requesting for a bound-with record (more technically, a constituent record).

  1. Retrieve availability of the constituent record (the record the user is looking at) using retrieve bib Alma API. (There will be a holding directly attached to the constituent record when there is a version of that item that isn't bound with anything)
  2. Retrieve availability of the MMS ID(s) in the 773 |w fields. (this is the mms_id of the host record)
  3. Combine both availability responses in the display.
Christina Chortaria at 2021-01-22 16:51:08 :
  • Are we going to add any logic in the traject file during indexing to get the location ?
  • or are we going to do this on the fly when we load the record page for the constituent record?
hackartisan at 2021-01-22 15:31:53 :

9962646063506421 could be used as a fixture for this process and more can be created if needed.

hackartisan at 2021-01-22 15:49:30 :

For go-live we will only worry about indexing the MMS ids of the host record in this field. Note there may be multiple host ids. (fixture needed)

For go-live we will only add host holding during availability calls. It will mean these records can't be found during an advanced search by location or faceting by location. This is a small service degradation and acceptable until we develop a plan for indexing.

Talking to @mzelesky - To do this during a full index would require 300k extra API calls.

Brainstorm of alternatives: - Do this during a 2nd pass on indexing where you find all the constituent records and add the holdings from their host records. This would require investigating turning on partial updates for solr. - If we could know that we always have the host records indexed first, then we could pull the holdings from them each time we hit a constituent record. - We could manage 2 sets of dumps: (non-bound-with and host records) and constituent records; or host records and (non-bound-with and constituent records).
