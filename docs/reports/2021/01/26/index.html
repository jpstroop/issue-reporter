mzelesky at 2021-01-26 15:10:41 :

So this is an example where there is a 950 field in the actual bib record that conflicts with the 950 created by the publishing enrichment.

I think the only real way to tackle this is to create a normalization process for catalogers that removes all fields from the bib record upon saving that the publishing job adds (950, 876, 852, 866, etc.), then run a periodic cleanup job on the records to remove those fields from records that missed the normalization.

So if there are traject errors related to unparseable fields (like the above fields), the best we can probably do is skip the record and report it out somehow. Otherwise you would have to add a lot of checking to remove fields that shouldn't be there.

One final note on this: it is possible to mark suppressed records as 'deleted' in the leader for the publishing job. This would be a more reliable way to exclude suppressed records than searching for a 950 field. This particular record is suppressed.

hackartisan at 2021-01-26 15:14:17 :

Okay skip it and report it is what we've done here in #1002 so I'm going to close this issue. The suppressed discussion should take place on #824. We'll talk about it at checkin today and then I will update that issue.
