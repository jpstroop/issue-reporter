#1220 Post incremental ReCAP dump files to S3

Creator:
hackartisan
Created:
2021-04-08 15:12:26
Status:
Open
Comments:
Trey Pendragon at 2021-04-13 17:58:14 :

Leaving some notes here in case I don't finish this in time:

Current way this works (a lot of this is going to end up deleted, and a lot of it is not tested):

  1. marc_liberation:recap_dump gets called by a rake task which is now deleted (https://github.com/pulibrary/bibdata/blob/50b293cd5fcdb02ff8716984c6f953c6a24ecc14/lib/tasks/sync.rake#L8-L10)
  2. Dump.dump_ids called with the dump type PRINCETON_RECAP (https://github.com/pulibrary/bibdata/blob/50b293cd5fcdb02ff8716984c6f953c6a24ecc14/app/models/dump.rb#L51-L53)
  3. A Dump is created with update_ids set to all the updated barcodes from the queries. NO files are created! https://github.com/pulibrary/bibdata/blob/50b293cd5fcdb02ff8716984c6f953c6a24ecc14/app/models/dump.rb#L99-L104
  4. Several slices of dump files are created. These are empty records, no actual file is saved yet. RecapDumpJob is called to generate the actual files associated with those DumpFiles: https://github.com/pulibrary/bibdata/blob/50b293cd5fcdb02ff8716984c6f953c6a24ecc14/app/models/dump.rb#L38-L45
  5. Each file which gets created at the DumpFile's path is uploaded to the S3 bucket (https://github.com/pulibrary/bibdata/blob/50b293cd5fcdb02ff8716984c6f953c6a24ecc14/app/jobs/recap_dump_job.rb#L17-L21). The files are just the default named DumpFile paths, which are named after the generation timestamp.

With the new process:

  1. Alma creates incremental ReCAP dumps and fires a message (done)
  2. AwsSqsPoller downloads the files, creates a Dump, and a DumpFile for every file that Alma outputs (done)
  3. AlmaDumpTransferJob should queue a transfer job for each DumpFile. (Probably a new job - RecapDumpJob no longer applies anymore. It could happen as part of AlmaDumpTransferJob, but I don't want the job to fail because the S3 bucket fails to connect or something.)

To test this we'll have to create an AWS bucket for alma-staging, which may be tough because that's currently set to be a production environment. Probably need some sort of flag for the bucket choice, or add another environment?

Edit: Don't worry about environment, just change the ENV variables in Ansible here: https://github.com/pulibrary/princeton_ansible/blob/e82313ee67ab3ebe949cdf3fdee1ef1bcd4920ca/group_vars/bibdata/alma_staging.yml#L118-L121
