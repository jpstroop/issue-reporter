Status:
Closed
Comments:
hackartisan at 2021-04-14 13:18:11 :

We held some meetings, with notes recorded in the document linked above.

The decision was to move away from a database and to a json file in the repository that holds all the location data.

Any time a change is needed to the location data (labels, flags, mappings), an issue should be created and a PR issued for the change.

the locations gem should load the data into memory and deliver API responses from here instead of from a database.

Similarly, traject should load the data into memory for use in translation maps.

These migraiton steps will be ticketed separately; closing this issue.
