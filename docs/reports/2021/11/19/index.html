Creator:
hackartisan
Created:
2021-11-18 20:17:45
Status:
Closed
Comments:
Trey Pendragon at 2021-11-19 20:22:47 :

I checked the JSON to see what could be erroring: https://figgy.princeton.edu/catalog.json?utf8=%E2%9C%93&q=Lapidus_11426331-02

There were two results - the second one had an ID that's not in postgres. ID: 76364993-48d3-40b7-91eb-3c13d88a4427

I searched the logs, and it looks like this corresponds to https://app.honeybadger.io/projects/53391/faults/82636440.

In the logs that corresponds to this error:

Task queue processing has stalled for 20001 ms with 0 remaining elements to process.

We saw this in reindexing.

I think what's happening is Solr's taking 20 seconds (which is NUTS!) to commit some big record combos. The indexing adapter sees this error and rolls back, with the assumption that solr's errored so it didn't get the record persisted. Unfortunately, Solr's throwing this error but -still indexing.-

I'm marking this sudden priority. If we can't fix this we might have to go back to solr 7.

The stallTime error is new in Solr 8.4.

Trey Pendragon at 2021-11-19 00:18:29 :

I'm hoping https://github.com/pulibrary/princeton_ansible/pull/2575 will fix this. I've deployed it, and I'll resolve the relevant honeybadger errors and we can keep an eye on it.

I deleted the one bad solr record, I'm going to see about doing a quick check for some bad solr records which would come up in searches.

Trey Pendragon at 2021-11-19 00:41:20 :
