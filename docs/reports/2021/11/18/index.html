I searched the logs, and it looks like this corresponds to https://app.honeybadger.io/projects/53391/faults/82636440.

Task queue processing has stalled for 20001 ms with 0 remaining elements to process.

I think what's happening is Solr's taking 20 seconds (which is NUTS!) to commit some big record combos. The indexing adapter sees this error and rolls back, with the assumption that solr's errored so it didn't get the record persisted. Unfortunately, Solr's throwing this error but -still indexing.-

I'm marking this sudden priority. If we can't fix this we might have to go back to solr 7.
