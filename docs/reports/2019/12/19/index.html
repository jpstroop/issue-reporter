#171 deploy is broken

Creator:
hackmastera
Created:
2019-12-18 21:38:02
Status:
Open
Comments:
hackmastera at 2019-12-19 18:45:36 :

There is conflicting documentation about the environment configuration for the upconfig and other commands.

https://lucene.apache.org/solr/guide/7_7/solr-control-script-reference.html states that -z is optional if ZH_HOST is set in solr.in.sh.

But https://lucene.apache.org/solr/guide/7_7/command-line-utilities.html#link-a-collection-to-a-configset states that -z is mandatory: -z or -zkhost <locations> ZooKeeper host address. This parameter is mandatory for all CLI commands.

So I think we need to just include it in the cap tasks.
