I don't think we should preserve ephemera boxes for two reasons: 1. So I expect that users would want to access a particular folder or set of folders, not everything in box 35.

They need to know the box number and/or barcode to retrieve the physical item. So preserving the box membership in the folder metadata would address that. Does that happen now?

It's pretty cheap to preserve metadata-only boxes, and if Figgy explodes we'll need it to recreate the Figgy that existed before that right?

Esmé Cowles at 2019-10-04T19:57:15:

@tpendragon That is true — we should preserve the boxes so we can reconstruct the box membership. I was conflating storing the boxes with storing folders nested in them, which is a different thing.
