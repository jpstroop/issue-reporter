#421 3 /24 hour Reserves Should not have a Request button

Creator:
carolyncole
Created:
2019-10-16T20:30:20
Status:
Open
Comments:
Kevin Reiss at 2019-10-16T22:18:12 :

If you look at the request form for the first bib ID:

https://catalog.princeton.edu/requests/1782308?mfhd=4056129&source=pulsearch

You'll see the 3-hour reserve copies are not requestable on that form. The issue lies somewhere in the OL codebase that decides whether or not to display the request button. The permanent location for these is "res" which is an offsite annex location where the button should be displayed so I suspect the OL code is missing something and not considering the temporary location assignment in deciding whether or not to display the button. If you look at the "data-requestable" attribute in the element with the selector location-services.service-conditional that decides if a button is shown in when the page loads. An ajax call in the availability.js script is supposed to look at the bibdata availability route results and decide if any of the default button behaviors need to be overridden based on a item's status or temporaray location assignment. Likely something is going wrong here: https://github.com/pulibrary/orangelight/blob/master/app/assets/javascripts/availability.js.coffee.erb#L120.
