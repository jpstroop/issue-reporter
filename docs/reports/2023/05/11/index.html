Testing strategy:

  1. An object where we've corrupted the local isilon file a. Run cloud fixity check on the FileSet with CloudFixity::FixityRequestor.queue_resource_check!(id: {fileset_id}). Result: success b. Run local fixity check on the FileSet with LocalFixityJob.perform_now({fileset_id}). We saw it enqueue and perform the RepairLocalFixityJob. Downloaded the file before and after. Result: successful repair.
  2. An object where we've replaced the cloud file with a different tiff a. Run local fixity check. Result: success b. Run cloud fixity check. Result: success
  3. An object where we've both corrupted the local isilon file and replaced the cloud file with a different tiff a. run cloud fixity check (which kicks off a local fixity check as well). Result: both fail, as expected; i.e. success.
