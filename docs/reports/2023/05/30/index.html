#3586 Investigate increasing solr app server resources

Creator:
Jane Sandberg
Created:
2023-05-26 15:21:53
Status:
Open
Comments:
Jane Sandberg at 2023-05-30 19:39:16 :

How much of the index can reside in memory?

du -h /solr/data/ | grep catalog # our collection takes up 79 gigs of disk
All solr collections together are 175 GB.
free -h # the boxes only have 38 gigs of RAM

So, if solr only put the catalog in memory and left all other indexes on disk, it could still only fit half of the catalog into memory. It is likely putting parts of the other cores into memory too, so probably only a small portion of the catalog collection is in memory

Do these boxes go to disk frequently?

This information can be found in our Solr Health Dashboard

In general, the catalog collections have a high (good) filter cache hit ratio and query result cache hit ratio. The document cache hit ratio is much lower (about 12%). The document cache evictions seems very high to me (97,234,000 in one week). According to the solr documentation, "The size for the documentCache should always be greater than maxresults times the maxconcurrent_queries, to ensure that Solr does not need to refetch a document during a request." We have the size set to 4096, but according to my understanding, this should actually be:

100 (maximum results/page available in the UI) x 300 (maximum concurrent requests available in passenger across the 3 app servers) + 1 (because it is supposed to be greater) = 30_001

This disregards the fact that users can enter a custom per_page in the search results URL, and also that sitemaps request more than 100 rows at a time.

How is CPU usage?

I looked at the Datadog metrics explorer, and it seems very reasonable on production. It seems over-resourced on staging.

How is disk usage?

Plenty of space

What storage medium is used?

pulsys@lib-solr-prod4:~$ lsblk -o name,rota
NAME               ROTA
sda                   1
└─sda1                1
  ├─lib--vg-root      1
  └─lib--vg-swap_1    1
sdb                   1
└─sdb1                1
  └─lib--vg-root      1
sdc                   1
└─sdc1                1
  └─lib--vg-root      1
sr0                   1

A rota of 1 indicates that it is a rotational device, and hence, not an SSD.

Recommendations

  1. Substantially increase the amount of RAM available to the catalog collection
  2. Increase the document cache size
