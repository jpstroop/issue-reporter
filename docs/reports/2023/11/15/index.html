#4982 DaoUpdater failed to update resource (a clearinghouse)

Creator:
Trey Pendragon
Created:
2022-02-10 15:54:17
Status:
Open
Comments:
hackartisan at 2023-11-15 15:17:55 :

Ways to tell if a resource has a DAO (I think): - in figgy: it will have to have an ark and be complete. - in aspace, the DAO shows as a "digital object" section - in pulfalight: go to the /raw version of the page and check the value of direct_digital_objects -- this gets populated based on the presence of a dao in the ead. - if there's a dao in aspace but not pulfalight then it maybe hasn't been reindexed yet. See below for getting the xml from aspace and looking at it directly.

look at the xml directly from aspace:

  • In the /raw solr record of the aspace object, copy the value of collection_unitid_ssm. Use this as the first value below (collection_id).
  • In the arclight project directory, open a rails console.
collection_id = "repositories/3/resources/1516"
uri =  collection_id.gsub("resources", "resource_descriptions")
File.write("scratch.xml", Aspace::Client.new.get_resource_description_xml(resource_descriptions_uri: uri, cached: false))

In vim, open the scratch.xml file. Then highlight all the contents of the file and paste ! xmllint --format - and hit enter. Note this depends on having the xmllint utility installed.

Now you can search for the number portion of the id found in system_identifier_ssm and look below that place in the hierarchy for a dao tag.
