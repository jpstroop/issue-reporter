One of the things that I've learned while working on #901 is that AWS S3 does not really have the concept of folders, instead S3 sort of fakes them with prefixes.

Based on the documentation here I wouldn't be surprised if AWS does not really support copying a folder as we envision it.

When you use the console to copy an object named with a trailing /, a new folder is created in the destination location, but the object's data and metadata are not copied.

The information in the aforementioned link is for the AWS console but the interesting bit is that folders (i.e. objects that end with /) work a bit different from other objects.

Information about AWS S3 folders in general that seems relevant to this issue: https://docs.aws.amazon.com/AmazonS3/latest/userguide/using-folders.html

Amazon S3 has a flat structure instead of a hierarchy like you would see in a file system. However, for the sake of organizational simplicity, the Amazon S3 console supports the folder concept as a means of grouping objects. It does this by using a shared name prefix for objects (that is, objects have names that begin with a common string). Object names are also referred to as key names.

Below is an example to let S3 automatically create the folder and another that manually creates the folder and in both instances the 1969 date shows in the Globus File Manager:

    # References:
    # https://docs.aws.amazon.com/AmazonS3/latest/userguide/using-folders.html
    # https://docs.aws.amazon.com/AmazonS3/latest/userguide/upload-objects.html
    key_id = ENV["AWS_S3_KEY_ID"]
    secret_key = ENV["AWS_S3_SECRET_KEY"]
    region = ENV["AWS_S3_PRE_CURATE_REGION"]
    bucket = ENV["AWS_S3_PRE_CURATE_BUCKET"]
    s3_credentials = Aws::Credentials.new(key_id, secret_key)
    s3_client = Aws::S3::Client.new(region: region, credentials: s3_credentials)

    # Example 1 - let S3 automatically create the folder
    #
    # Upload the file to a "folder" named "hector-fake-folder-four"
    # The "folder" is automatically created with date 1969
    file_key = "hector-fake-folder-four/feb-1-405.txt"
    file_name = "/Users/correah/Downloads/small.txt"
    options = {client: s3_client}
    object = Aws::S3::Object.new(bucket, file_key, options)
    object.upload_file(file_name)

    # Example 2 - create the folder manually ahead of time
    #
    # Force the creation of a folder (notice the trailing / in the file_key)
    # The folder will have the 1969 date.
    file_key = "hector-fake-folder-six/"
    options = {client: s3_client}
    object = Aws::S3::Object.new(bucket, file_key, options)
    object.put({content_length: 0})

    # ...and the upload a file to it
    #
    file_key = "hector-fake-folder-six/feb-1-409"
    file_name = "/Users/correah/Downloads/small.txt"
    object = Aws::S3::Object.new(bucket, file_key, options)
    object.upload_file(file_name)
