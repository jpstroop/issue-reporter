#8 migrate images

Creator:
Cliff Wulfman
Created:
2023-10-05 12:48:13
Status:
Open
Comments:
Cliff Wulfman at 2023-10-05 12:50:51 :

We received a zip archive containing 21,986 image files from OIT. Unfortunately, the file names do not match those in the tables; MIchael Muzzie did some investigating and discovered an earlier developer had added some complexity:

Each image in the imagetable has the filename and the bookid and the imagecounter fields. Then each book in the booktable has a bookid and a bookrandomid. The two tables relate to each other via the bookid, and then the uploaded filename got re-saved as bookrandomid underscore imagecounter underscore underscore then o, d, or t (original, downsize, or thumbnail).

So the original PrinUL O-123alt.jpg is actually FsJBnas0oKkTfhlk1o.jpg PrinUL O-123alt2.jpg is FsJBnas0oKkTfhlk2o.jpg DSC0007.JPG is FsJBnas0oKkTfhlk4o.jpg And so on…

Cliff Wulfman at 2023-10-05 14:41:42 :

I created a mapping of old names to new and added it to the Husby Data spreadsheet: https://docs.google.com/spreadsheets/d/1K7o0gOKXOZnrIJxhLlAFPYUvcBWQSiO_7Fw2rJk52Bc/edit?userstoinvite=mdmuzzie@princeton.edu&sharingaction=manageaccess&role=writer#gid=1398527934

Unfortunately, they weren't all mapped. Investigating.

Cliff Wulfman at 2023-10-05 15:03:16 :

There are duplicate filenames in the image table. In some cases, the same image is mapped to several entries in the book table. Seems odd.
