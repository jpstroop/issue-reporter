  • There's nothing useful in /var/log/redis/redis-server.log on lib-redis, not even a line about the error. ~~- We have two projects that use Lua (that I can find): specialcollections and pullibrary_drupal.~~ ~~- This SO post suggests we could modify the global lua-time-limit for the Redis server (default 5,000 ms). There's no current entry for this config setting in /etc/redis/redis.conf on lib-redis.~~
  • However, this article defines the lua-time-limit as the amount of time Redis rejects client commands to allow a script to complete. After this limit is reached, if the script continues to execute, Redis accepts client commands but returns the BUSY error. So updating the lua-time-limit won't fix the problem we're seeing.
  • This GitHub issue suggests we could increase the Redis client timeout (default 5 seconds), like this: Redis.new timeout: 30. ~~- Alternatively, we could try to optimize the Lua scripts to run more quickly.~~
  • Looking at the output of SLOWLOG GET on the redis server shows several entries. They are not Lua scripts, though, they are queries from library-prod. The output is not easy to parse, but if we need to dig deeper, this is a place to start. See https://bobcares.com/blog/redis-slow-log/ and https://docs.redis.com/latest/rs/clusters/logging/redis-slow-log/.
