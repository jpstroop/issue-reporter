pulibrary/pulfalight

#1117 Ongoing issues with Sidekiq/Redis

Creator:
Eliot Jordan
Created:
2022-09-02 15:49:38
Status:
Open
Comments:
Alicia Cozine at 2023-01-11 22:31:43 :

Notes from some digging today:

  • The error message seems to indicate that a long-running Lua script is still in progress on the Redis server when pulfalight is trying to connect at these times. So whatever is happening is likely happening on lib-redis, not on the pulfalight boxes.
  • There's nothing useful in /var/log/redis/redis-server.log on lib-redis, not even a line about the error.
  • We have two projects that use Lua (that I can find): specialcollections and pullibrary_drupal.
  • This SO post suggests we could modify the global lua-time-limit for the Redis server (default 5,000 ms). There's no current entry for this config setting in /etc/redis/redis.conf on lib-redis.
  • However, this article defines the lua-time-limit as the amount of time Redis rejects client commands to allow a script to complete. After this limit is reached, if the script continues to execute, Redis accepts client commands but returns the BUSY error. So updating the lua-time-limit won't fix the problem we're seeing.
  • This GitHub issue suggests we could increase the Redis client timeout (default 5 seconds), like this: Redis.new timeout: 30.
  • Alternatively, we could try to optimize the Lua scripts to run more quickly.
