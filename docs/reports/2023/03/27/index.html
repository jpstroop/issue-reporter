#5669 Local fixity only runs on primary file

hackartisan
Created:
2023-02-09 14:09:07
Status:
Open
Comments:
Eliot Jordan at 2023-03-27 16:39:28 :

After looking into this, I recommend that we don't change the behavior of the local fixity job. Local fixity is checked on the primary file: https://github.com/pulibrary/figgy/blob/main/app/jobs/local_fixity_job.rb#L94

which is one of either original_file, preservation_file, or intermediate_file: https://github.com/pulibrary/figgy/blob/main/app/models/file_set.rb#L93

Similarly, the cloud preserver creates binary nodes for each of those file types: https://github.com/pulibrary/figgy/blob/main/app/services/preserver.rb#L75

However, it seems to me that there is always only one primary file type per FileSet. Is this true?

Regardless, I feel that checking fixity only on the primary file is sufficient.

Thoughts?
