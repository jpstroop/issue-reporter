#2119 Indigenous peoples of Mexico not masking correctly consistently

Creator:
Max Kadel
Created:
2023-03-31 19:06:22
Status:
Open
Comments:
Max Kadel at 2023-04-03 14:43:54 :

One trend I may be seeing is that the incorrect subjects seem to have multiple subfield z's in them, and I'm not seeing that with the records that have correct subjects, but I haven't done a systematic analysis, just a quick survey.

Max Kadel at 2023-04-03 14:52:52 :

An example that is on both production and staging / sandbox environments is https://catalog.princeton.edu/catalog/99125398364906421 / https://catalog-staging.princeton.edu/catalog/99125398364906421

Max Kadel at 2023-04-03 16:20:15 :

I think the issue is with the gsub in this line, when the geographic subdivision repeats a term that is already in the main subject term.

Record xml: xml <datafield tag="650" ind1=" " ind2="0"> <subfield code="a">Indians of Mexico</subfield> <subfield code="z">Mexico</subfield> <subfield code="z">Hidalgo (State)</subfield> <subfield code="x">Antiquities</subfield> <subfield code="v">Congresses.</subfield> </datafield>

Bibdata code: ruby heading = heading.gsub(" #{s_field.value}", "#{SEPARATOR}#{s_field.value}") if split_on_subfield.include?(s_field.code)

At this point in the code: ```ruby heading = "Indians of Mexico Mexico Hidalgo (State) Antiquities Congresses." sfield.value = "Mexico" sfield.code = "z" splitonsubfield.include?(s_field.code) = true

so what the code is trying to do is

heading = "Indians of Mexico Mexico Hidalgo (State) Antiquities Congresses.".gsub(" Mexico", "#{SEPARATOR}Mexico"

so it's substituting both instances of " Mexico" with "—Mexico", so

heading = "Indians of—Mexico—Mexico Hidalgo (State) Antiquities Congresses."

Then the rest of the subfields are substituted in the same way, with the final result

heading = "Indians of—Mexico—Mexico—Hidalgo (State)—Antiquities—Congresses" ```
