hackartisan at 2023-04-14 21:04:04 :

https://github.com/pulibrary/figgy/pull/5750 updates manifests as per the implementation proposal in the previous comment. However, UV only implemented the rtl direction feature for the metadata info pane, not for the leftPanel "Contents" iiif-tree-component.

Code references:

  • the TreeView in the leftpanel module uses TreeNode for the elements of the table of contents https://github.com/UniversalViewer/universalviewer/blob/86f2cd47598ed2b2cef6b30fd3dc706b0c8b9807/src/content-handlers/iiif/modules/uv-contentleftpanel-module/TreeView.ts
  • The TreeNode itself is defined in manifesto: https://github.com/IIIF-Commons/manifesto/blob/6e355e34c38b64b9cf9c366d6ff60006db341968/src/TreeNode.ts
    • the label is set in the constructor, making it much more difficult to override the behavior here, to the extent that it's either not actually possible or really not a good idea.
  • Here's some code showing how it works in the metadata info pane: https://github.com/UniversalViewer/universalviewer.github.io/blob/85dac1a4274f53659ba136c2bcd7ce1ab8c9d908/vendor/uv/lib/iiif-metadata-component.js#L375-L382
  • language direction configuration referenced in the UV ticket: https://github.com/UniversalViewer/universalviewer/blob/67ada49f6d866a51ab173d7166f6ddf04fffb471/src/content-handlers/iiif/extensions/uv-mediaelement-extension/options/en-GB.json#L50

Note that @tpendragon also tried just putting html as the manifest value, but that resulted in an error.

The proposed path forward is (not necessarily in this order): * Work on UV upstream to implement the required feature, possibly in collaboration with someone in the UV community. * Migrate from webpacker to vite (https://github.com/pulibrary/figgy/issues/5734) * Update figgy to use the latest UV release, ensuring we carry over local customizations

However, this path forward is a bigger project than we had hoped and the work will need to undergo prioritization again.
