Trey Pendragon at 2023-04-19 16:02:38 :

@maxkadel For documentation, https://github.com/pulibrary/orangelight/pull/3286 is the PR where I broke this.

I think it's this:

If I get back two resources and one of them is in the member_ids of another, don't display the child.

There's never a case where we want to display the MVW manifest -and- the volume manifest on the same page.

We used to filter it out so if there was a MVW and a Volume then it'd only display MVWs, but finding if something is a MVW is pretty expensive and that isn't a precise enough calculation anyways.

Max Kadel at 2023-04-19 19:52:09 :

@tpendragon That PR is part of why I'm confused.

Scenario 1 - 1 physical object with 2 Figgy records, one for regular imaging, one for multi-spectral imaging

So, https://catalog.princeton.edu/catalog/9950403683506421 should still have 2 viewers after whatever changes I make, since there is * 1 physical object with a recto and verso * 1 catalog record for that object * 2 Figgy objects * 1 Figgy object is a simple object (not nested) with 2 images, one of the recto and one of the verso, which translates to one iiif viewer (see https://figgy.princeton.edu/catalog/bbc6f6c4-3b92-4ae9-8461-1a14c113af8c) * This is represented in the iiif manifest as @type: "sc:Manifest" * 1 Figgy object is a complex object with two nested "members" (see https://figgy.princeton.edu/catalog/54b399c7-f28c-46f6-a7b4-c835a60516c4) * Each "member" is a set of multi-spectral images * Both members are represented in a single iiif manifest as @type: "sc:Collection", resulting in a single viewer

Scenario 2 - 1 map in 6 sheets and 1 Figgy record

https://catalog.princeton.edu/catalog/9968107933506421 - Should have a single viewer after whatever changes I make * 1 conceptual object in six pieces (1 map, 6 sheets) * 1 catalog record * 1 Figgy object * Complex object with 7 nested "Child Scanned Maps" (one for each sheet, plus one of all of the sheets digitally stitched together) * Each manifest that Orangelight fetches has @type: "sc:Manifest". The one that has all of the images has multiple canvases under Sequences: Canvases, whereas the single images on have on canvas.

So, could we say that the number of viewers should match the number of (parent?) Figgy objects?
