Status:
Open
Comments:
Kevin Reiss at 2023-12-13 15:13:58 :

Atlas tech support recommends we do the following. Update our code to only submit the following params a single time in a given post:

AeonForm DocumentType GroupingIdentifier GroupingOptionCallNumber GroupingOptionItemDate GroupingOptionItemInfo1 GroupingOptionItemInfo3 GroupingOptionItemInfo4 GroupingOptionItemNumber GroupingOptionItemTitle GroupingOptionItemVolume GroupingOptionLocation GroupingOptionReferenceNumber GroupingOption_Site Location RequestType SubmitButton SystemID UserReview WebRequestForm

Currently we submit these params for every container included in a given POST request. Additionally they recommend the values for these two params should be set to the following values: AeonForm=EADRequest WebRequestForm=EADRequest

AeonForm tells Aeon that the POST contains EAD data, WebRequestForm tells Aeon which aeon request form the requests submitted should be edited in within the Aeon web interface if the end user decides to do that.
