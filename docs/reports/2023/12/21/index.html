#3 Broken vertical bar character causes error when using Chinese (Taiwan) locale

Creator:
tventimi
Created:
2023-12-14 13:02:36
Status:
Closed
Comments:
tventimi at 2023-12-21 17:05:38 :

Fixed in release v1.1.1. Hard-coded instances of the broken bar character were replaced with ChrW(166). Also, the registry object had to be created in a different way, using GetObject("winmgmts:{impersonationLevel=impersonate}!\.\root\default:StdRegProv") and its methods GetStringValue and SetStringValue. This is because broken bars were being used as separators in the field sets and URL lists, but the Chinese locale was converting them to solid bars. This was only happening when using WScript.Shell/RegRead/RegWrite, but not with the new implementation.
