Response from OIT:

This problem is typically caused by applications that redirect back to themselves multiple times, adding more cookie data or GET URLs each time. We've already increased the maximum headers size allowed by the SSO systems as much as we dare, but the problem still happens occasionally. Clearing browser caches or using new/private/incognito browser instances may temporarily work around the problem, but we cannot prevent all applications from misbehaving. At best, we can ask vendors to fix their applications.
