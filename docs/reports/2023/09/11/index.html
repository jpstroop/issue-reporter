#15 Communicate directly with solr for catalog and dpul

Creator:
Jane Sandberg
Created:
2023-09-11 19:28:06
Status:
Open (pull request)
Comments:
Christina Chortaria at 2023-09-11 19:59:17 :

What is the best way to set up a dev environment? Create an SSH tunnel? Or have a bunch of solr collections running locally in lando?: For a dev environment it would be better to use lando and have solr colelctions running locally.

Max Kadel at 2023-09-11 20:18:46 :
  • Why is the findingaids blacklight API response so different than the solr API response? Which is preferable?

I don't know exactly why the sort is so different, but my best guess that it has to do with the CatalogController and sort behavior.

I think it's important that the sort match between the Allsearch and the specific service's default search, because I think otherwise it will be very disorienting for users. This also gives the service application developers more fine-grained control over search result sorting.

I understand the desire to hit the Solr API directly, for speed and load, but I think specifically re-creating the Solr queries the different Blacklight apps create will be time consuming, and would be very prone to drift. I think there's similar risk for breaking changes between the Blacklight apps and hitting Solr directly.

Jane Sandberg at 2023-09-11 21:00:12 :

specifically re-creating the Solr queries the different Blacklight apps create will be time consuming

Almost all of the complexity in what Orangelight sends to Solr is related to facets, which we wouldn't need to recreate for this application. Of the 52 lines in this query, for example, only one has to do with the sort order. And I'd be happy to add a sort param to my PR.

Not sure if this is true of the other applications as well.

Kevin Reiss at 2023-09-11 21:21:51 :
What is the best way to set up a dev environment? Create an SSH tunnel? Or have a bunch of solr collections running locally in lando?

Creating four local local collections in solr for all four of these applications seems like a lot to manage and build. For local dev purposes couldn't we just query the staging or qa environments (regardless of using solr or the blacklight api) of these applications when the application is running in dev mode.
