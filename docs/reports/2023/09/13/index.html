#15 Communicate directly with solr for catalog, dpul, and pulfalight

Creator:
Jane Sandberg
Created:
2023-09-11 19:28:06
Status:
Closed (pull request)
Comments:
Kevin Reiss at 2023-09-13 04:50:55 :

In terms of arriving at a decision on Blacklight API vs. Solr since we have implemented both talking to the Blacklight API and talking directly to solr can we compare the following points:

  1. Response Time
  2. Bandwidth saved by not returning data related to facets

The current discoveryutils application has been rather stable over the years. There has not been a lot of drift or unexpected display issues that have popped up due to solr schema changes. The only major work has been to handle major version upgrades of Blacklight apps when the catalog.json API changed radically. We had to make some adjustments for the catalog piece when we moved to Alma to handle changes to how URLs were represented.

Jane Sandberg at 2023-09-13 16:35:31 :

I benchmarked this branch against main using the following shell script: ```

!/bin/sh

declare -a words=("check" "quaint" "flamingo" "piano" "team" "hiccup" "cherry" "literature" "nomination" "diameter" "layer" "mutter" "tongue" "inspire" "amber" "rugby" "rescue" "width" "brush" "braid") for word in "${words[@]}" do curl "http://localhost:3000/search/catalog?query=$word" done ```

Here's how long it took (in seconds) to run this script for each endpoint:

Endpoint Solr backend (this branch) Blacklight backend (main)
/search/catalog 4.974 24.649
/search/dpul 4.980 19.726
/search/findingaids 4.960 20.184

Depending on the endpoint, this branch is 4-6x faster than main.

Message for stakeholders

The DACS team is excited to invite you to test a new release of the catalog in our catalog-qa environment. This release of Orangelight is focused on improving the application's performance and ensuring the catalog is compatible with newer versions of the Solr indexing server we use to index our catalog data. The release strives to implement the same search functionality the current production catalog has for basic and advanced search features.

The new release will be available for the new two weeks in the QA environment. To test you need to do the following.

  1. Connect to Global Protect if you are off campus or on on campus wifi.
  2. Visit https://catalog-qa.princeton.edu.
  3. Utilize the system, particularly the basic and advanced search features.
  4. Please report errors or questions to this email thread or in slack on either the catalog or orangelight channels.
  5. When reporting an error please try and capture the URL and a screenshot (if possible) where you encountered something unexpected. Try and describe the issue in as much detail as you can.

Thanks for assisting with evaluating this release,

Regards, DACS Team
