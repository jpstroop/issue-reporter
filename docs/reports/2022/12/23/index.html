Alicia Cozine at 2022-12-23 16:51:31 :

Ah, yes, this is a precedence issue. See https://docs.ansible.com/ansible/latest/reference_appendices/general_precedence.html for the rules of precedence for other settings besides variables.

You can pass -e ansible_user=pulsys to override the entry in the playbook. We could add a comment to the playbook suggesting this as a workaround.

Alternatively, we could change that line in the playbook to be remote_user: "{{ connection_user }}" and then set a default in the role of tventimi. If we did that, you'd still have to pass -e connection_user=pulsys to run as the pulsys user, though. Would that make it easier? More obvious what to do?
