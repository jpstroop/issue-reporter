It seems that the Books table is populated with data from the public/cicognara.mrx.xml (or is it public/cicognara.marc.xml?) and there is indeed no match for the missing records in this MARC XML file.

This is the code that process the MARC XML: https://github.com/pulibrary/cicognara-rails/blob/main/lib/cicognara/tei_indexer.rb#L88 -- it is possible that the Books table was populated with the BookIndexer but it is not clear to me how to trigger this code.

Notice that just adding the problematic DCL numbers to the catalogo.tei.xml file as indicated here does not seem to fix the issue since the match is not found on the Books table.
