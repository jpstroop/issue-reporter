Matthew J. Chandler at 2022-11-15 14:05:04 :

It looks like the quotation marks were stripped out of the title (they should remain, as the dataset title includes a reference to the paper title). The publisher should be Princeton University. The description field appears to have mashed together two different description fields from the original record (dc.description.abstract and dc.description). If the original record includes multiple descriptions, I think they should be given as separate elements in DataCite, but I'm not actually sure if we want to carry over the second field in this case, given its reference to a Globus link (@hhkitsune what's your take on that?). The text in the description (from the original abstract) also seems to have a character encoding error for an apostrophe in "Earth's", and the quotation marks from the paper title are stripped out (they should be there).

Hannah H at 2022-11-15 14:45:10 :

@matthewjchandler We are putting the description elements into Datacite's descriptionType: Other. There should be a space between sections when more than one descriptionType is put together into "other." In the future, we can remediate the legacy data to be more specifically nuanced to appropriate categories of descriptionType. We are using this approach to scale the migration. Only a few of us have significant experience describing research data to know when to use the various categories, and these same persons are juggling other duties. If we put everything into one place it will easier to sort out, whereas if we have a bunch of categories and some of these are wrong, it will be more difficult to identify and correct. Please see me if you have further thoughts or questions about this approach.

Regarding the Globus link and explicit README statements, this is something we can talk about. My initial thought is that these aren't really needed and may have been statements created in DSpace. However, I'm wondering if these might be useful for someone using a reading device to access the information on the page. @matthewjchandler Do you know if this would be useful in such a case?
