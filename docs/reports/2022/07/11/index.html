Creator:
Kim Leaman
Created:
2022-07-08 15:10:59
Status:
Open
Comments:
hackartisan at 2022-07-11 14:27:05 :

The browse collection widget is taking a long time to query solr. You can see that the browse collection page itself loads quite slow. https://dpul.princeton.edu/lapidus/browse/browse-the-collection. One workaround would be to remove that widget from the exhibit home page. Another would be to reduce the default search results per page for the exhibit; that does make the widget load faster (though still not fast). You can see load time for the smaller set of search results by doing https://dpul.princeton.edu/lapidus/browse/browse-the-collection?per_page=10. Here's a datadog log search showing dpul prod solr query times: https://app.datadoghq.com/logs?query=service%3Asolr%20dpul%20-host%3A%28lib-solr-staging1%20OR%20lib-solr-staging3%29%20%40path%3A%22%2Fselect%22%20exhibit_lapidus_readonly_title_ssim&cols=%40QTime&index=&stream_sort=time%2Cdesc&from_ts=1657462094473&to_ts=1657548494473&live=true

query times are also slow for a blank search submitted through the search form.

Note the slow response is also observed in staging, https://dpul-staging.princeton.edu/lapidus and https://dpul-staging.princeton.edu/lapidus/browse/browse-the-collection
