Creator:

2021-12-15 16:54:32
Status:
Open
Comments:
Alicia Cozine at 2022-02-17 22:35:19 :

I started to look at this today, but my keys are not on the staging server yet.

On the production server, the logs look like they are getting rotated. There's one log per day and they are mostly timestamped at 23:59. There is no entry in /etc/logrotate.d for these logs, and no ezp.log file. The /var/local/ezproxy/config.txt file has some log directives in it. I'll pick this back up when I can and start with that config.txt file.

There's some minimal documentation at https://help.oclc.org/Library_Management/EZproxy/Configure_resources/LogFile.

Kevin Reiss at 2022-02-18 03:43:05 :

I think if the directives contain date time placeholders the software creates a new log every day. If we wanted to delete old log files I think we'd need to do something with logrotate.d or another scheduled cron task of some kind process. EZproxy doesn't seem to have any directives related to log retention. LogFile -strftime log/ezp%Y%m%d.log LogSPU -strftime log/spu%Y%m%d.log
