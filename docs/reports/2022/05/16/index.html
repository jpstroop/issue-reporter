Here's an index count validation strategy:

new_index_counts = Spotlight::Exhibit.all.map{ |e| [e.id, e.resources.count]}.to_h.sort
# Do the same on the machine connected to the old index and copy/paste it into `old_index_counts`
counts_zip = new_index_counts.zip(old_index_counts)
counts_zip.each do |tt|
  if tt[0][0] == tt[1][0] && tt[0][1] == tt[1][1]
    puts "e #{tt[0][0]} equal"
  elsif tt[0][0] == tt[1][0]
    puts "e #{tt[0][0]} new index count #{tt[0][1]} old index count #{tt[1][1]}"
  else
    puts tt
  end
end; nil

The full reindex timed at < 12 hours on staging with both machines (3 workers each). We should run it again with a single machine and double the workers.

hackartisan at 2022-05-16 14:18:50 :

To double the workers on one box:

  1. ssh to one box and sudo service sidekiq-workers stop
  2. ssh to the other box and sudo vim /etc/systemd/system/sidekiq-workers.service
  3. in the bundle exec sidekiq change the -c arg to double its value (from 3 to 6)
  4. run sudo systemctl daemon-reload
  5. sudo service sidekiq-workers restart
  6. observe the change in the sidekiq admin panel
