Alicia Cozine at 2022-05-02 16:43:58 :

We discussed possibly removing the update_cache: true line from most if not all apt tasks and moving that operation into a separate playbook or tasks file that does not run every time the common role runs.

When update_cache is set to true, Ansible tries to update the local index/list of packages and versions available from apt - the Ansible equivalent of typing sudo apt-get update. This is useful in general - it's a good idea to keep the apt index up-to-date. However, most of our apt tasks don't upgrade packages, so we don't absolutely need to update the cache before running most of them. And updating the cache does fail fairly often due to changing keys and other issues, which blocks other work from getting done.

Updating the cache is non-destructive and can be done on all infrastructure on a regular basis. So if we have a separate playbook for updating the apt cache, we can run it monthly or quarterly and fix any issues that come up.
