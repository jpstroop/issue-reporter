#4774 Resources updated in Pulfalight should update their metadata in Figgy automatically.

Creator:
Trey Pendragon
Created:
2021-11-05 19:52:46
Status:
Open
Comments:
hackartisan at 2022-05-03 17:55:28 :

endpoint /resources/refresh_remote_metadata expects a body like { source_metadata_identifiers: [...] } or { archival_collection_codes: [...] }. For this ticket implement the code path for archival_collection_codes.

Implementation

  • I get a list of things
  • I kick of a job for each thing in the list
  • That spawns a job per figgy resource that actually fetches the record/s from pulfalight
  • See if metadata changed somehow before we persist to prevent needlessly re-preserving.
    • Note that the source metadata sync happens in the change set persister; we would have to compare up front somehow
  • Use batches?
    • Would let us see the hierarchy of these jobs.

Example:

  • I update C0001_c01 in ASpace
  • Pulfalight hears that C001 changed.
  • Pulfalight reindexes the full collection, because it has to, that’s how Pulfalight works.
  • Pulfalight posts to figgy /resources/updatemetadata with body { archivalcollection_codes: [“C0001”] }
  • Figgy hears the request, spawns a job for updating per collection code, returns “202 Accepted”.
    • queue low priority
  • That job gets all resources which have that archival collection code, and spawns a metadata refresh job for each of those resources.
  • The metadata refresh job should try to update, if nothing changes then don’t persist.
    • This metadata refresh job should be in the low priority queue.
  • How many jobs per metadata update?
    • End up with one job per resource to be updated.
