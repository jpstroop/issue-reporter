Looked at this a bit more with @tpendragon. Notes:

For figgy to have malfunctioned in the manner observed, the AutomaticIngester would have to have moved the file to an incorrect location, then crashed the process without alerting honeybadger. If it moved to the correct location we would see the file there. And if it didn't crash then PDFIngestJob would still have been invoked.

https://github.com/pulibrary/figgy/blob/0d3d0dc7e380664a89e9b602e452495518a25ddd/app/services/cdl/automatic_ingester.rb#L20-L21

This scenario seems extremely unlikely. It seems more likely that some other process is moving the file before the ingester is run. The rake task that runs that ingester is scheduled to run each hour.

(Also note that there are some PDFIngestJobs from today; this issue is intermittent.)
