#5078 File appender failure notices are too frequent

Creator:
hackartisan
Created:
2022-04-19 15:02:10
Status:
Open
Comments:
hackartisan at 2022-08-11 16:17:33 :

One thing I notice is that even when there are failed derivatives for these resources, there's not the banner message "x/y derivatives generated" or whatever it says that cues ingesters to know there may be an issue. If they're looking for that and not scanning through the actual file list or viewer they won't notice these problems.

Error analysis.

  • "multipart upload failed" (note nested Net::OpenTimeout errors): Needed derivatives regenerated. Steps I took: Use the resource id to get to the resource in figgy, click through to its parent, go to file manager, click "regenerate derivatives" button for the corresponding file.
    • some of these were fine; may have been regenerated by ingester
  • Net::OpenTimeout
    • one was fine (although it may have been regenerated by ingester)
    • some others needed derivatives regenerated
  • "multipart upload failed: The difference between the request time and the current time is too large.": needed derivatives regenerated
  • "SSL_connect SYSCALL returned=5 errno=0 state=SSLv3/TLS write client hello": needed derivatives regenerated
  • "multipart upload failed: SSL_connect SYSCALL returned=5 errno=0 state=SSLv3/TLS write client hello": needed derivatives regenerated
  • "execution expired": needed derivatives regenerated
  • "multipart upload failed: Your socket connection to the server was not read from or written to within the timeout period. Idle connections will be closed.": needed derivatives generated; several of these
  • "multipart upload failed: Connection reset by peer": needed derivatives regenerated
  • "The AWS Access Key Id you provided does not exist in our records.": this stemmed from key rotation event but nothing needed to be regenerated. not sure if someone else did that.
  • "multipart upload failed: Failed to open TCP connection to iiif-image-production.s3.amazonaws.com:443 (getaddrinfo: Temporary failure in name resolution)"
    • needed to be regenerated

This one was not something I had seen before. - multipart upload failed: undefined method `read' for #IngestableFile:0x0000555f3e839558 - created https://github.com/pulibrary/figgy/issues/5352
