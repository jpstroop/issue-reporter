@kelea99 Barcodes used to be required for the Request feature bundling logic, and I believe they still are. Sometimes multiple components have the same barcode. Other times a single component has multiple barcodes. As long as PulfaLight has access to the barcodes and which components they are matched with, it can properly bundle requests for Aeon.

I'm not sure it matters where they live or how the data gets there. I think it probably would make sense to create some kind of macro or script for adding barcodes to ArchiveSpace from appendixainventory.csv so archivists don't have to add them manually.

This is inventory management and I'm of the opinion that is not appropriate to do this in EAD. My suggestion at the time it was implemented this way was to have a separate lookup system for inventory management, similar to what ReCap does, but with an API.
