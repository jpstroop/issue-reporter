#5369 Restore f61a4cf2-0d08-412b-8b33-93fce11069e0 from preservation storage

Creator:
Trey Pendragon
Created:
2022-08-17 16:47:24
Status:
Closed
Comments:
Trey Pendragon at 2022-08-17 16:51:03 :

A couple of things we discovered and talked about:

Figgy deletes files from preservation, because it's versioned, when it's deleted in Figgy. Keeps preservation looking pretty and up to date with what the current state of things is.

The google console doesn't display deleted resources. You have to use the Google Cloud Storage (GCS) CLI or Figgy's versioned_google_cloud_storage adapter to get a hold of them.

A UI or documented process for restoring a whole object from preservation storage is one of the tickets we never finished and will undertake for our preservation cycle.

We managed to restore this like this:

Preserver::BlindImporter.import(
  id: "f61a4cf2-0d08-412b-8b33-93fce11069e0",
  change_set_persister: ChangeSetPersister.default,
  source_metadata_adapter: Preserver::BlindImporter::FileMetadataAdapter.new(
    storage_adapter: Valkyrie::StorageAdapter.find(:versioned_google_cloud_storage
  )
)
