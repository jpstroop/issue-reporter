irb(main):004:0> User.find_by(uid: patron_id)

Max Kadel at 2022-06-29 17:28:01 :

This is looking like a potential avenue for investigation - https://github.com/pulibrary/orangelight/blob/2d915eaddeb22c1b8e171d3b9e68fd5cc01b6cea/app/models/requests/patron.rb#L198-L209 

      def build_alma_patron(uid:)
        alma_user = Alma::User.find(uid)
        barcodes = alma_user["user_identifier"].select { |id| id["id_type"]["value"] == "BARCODE" }
        {
          last_name: alma_user.preferred_last_name,
          active_email: alma_user.preferred_email,
          barcode: barcodes.first&.fetch("value") || "ALMA",
          barcode_status: 1,
          netid: "ALMA",
          university_id: uid
        }
      end

I think this is building the patron from the login response from Alma, rather than from the Bibdata response, and it looks like it's grabbing the first barcode, rather than the active barcode, which would match what we're seeing when the error happens.
