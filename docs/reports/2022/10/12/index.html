#3350 Upgrade lib_www role to PHP 8.1

Creator:
Kevin Reiss
Created:
2022-09-21 23:08:32
Status:
Open
Comments:
Jane Sandberg at 2022-10-12 21:09:59 :

@maxkadel and I investigated this at length today. Unfortunately, the Drupal 7 date_ical module relies on a very outdated version of the iCalcreator library, which is not compatible with PHP 8.

For context:

  • The date_ical module documentation says: "The author of iCalcreator made extenside backwards incompatible changes to the library in the v2.22 release. Thus Date iCal does not support any version of iCalcreator after v2.20.x."
  • PHP 8 support is not mentioned in the iCalcreator release notes until https://github.com/iCalcreator/iCalcreator/blob/master/releaseNotes#L132-L155

The results on PHP 8 are:

  • With iCalcreator 2.20.2 (the version currently in main), the libcal importer fails with an error about the curly brace array syntax (e.g. $array{0})
  • With iCalcreator 2.20.4, the libcal importer hangs for several minutes, before giving a database timeout error

#1171 Make autocomplete text invisible in search bar

Creator:
hackartisan
Created:
2022-09-27 14:29:45
Status:

Comments:
hackartisan at 2022-10-12 13:59:06 :

The CSS ways of doing this had undesirable accessibility side effects. Either we could hide the autocompleted text visually but not from screen readers, or we could effectively hide it but it also hid the search bar from screen readers.

It looks like there's a hint option on the typeahead initialization that is set to "true" in blacklight. It seems worth trying to override that initialization and set it to false instead.

https://github.com/twitter/typeahead.js/blob/master/doc/jquery_typeahead.md

I would try copying most of https://github.com/projectblacklight/blacklight/blob/v7.29.0/app/javascript/blacklight/autocomplete.js into pulfalight_loader.es6 to see if we can initialize it with the hint off.

Also, an interesting note, blacklight has removed twitter typeahead for version 8: https://github.com/projectblacklight/blacklight/pull/2576
