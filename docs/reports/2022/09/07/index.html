Sarah Hamerman at 2022-09-07 15:04:22 :

a couple of clarifications to Phoebe's messages (which were paraphrased from our slack conversation) :)

A couple of edits: The ASpace MARC exporter is generating 6xx $2 codes that are not valid. The main type is $2 viaf because we use viaf identifiers/URIs in ArchivesSpace but there isn't an approved $2 code in for viaf. The list of valid codes is here : https://www.loc.gov/standards/sourcelist/subject.html It also creates 6xx _7 $2 uncontrolled. The correct format would be 6xx _4 with no $2 for an uncontrolled heading: https://www.loc.gov/marc/bibliographic/bd600.html However, 6xx _4 doesn't show up in Blacklight, so we would really want to change them all to 6xx _0 even though some of the names are not valid LCSH We don't want to add 6xx _4 because there are a lot of other libraries' local vocabularies in various records that are coded that way. We do want to retain the $0 for the viaf headings.

Creator:

hackartisan at 2022-09-07 15:54:55 :

We investigated the object in question and found that it has 37 leaf structure nodes that hold proxy objects with ids for (FileSet) resources that have been deleted.

Relevant code: https://github.com/pulibrary/figgy/blob/0ee595585c6e94d686f29790084445892830797b/app/views/base/_structure_node.html.erb#L22-L25

The array holds the proxy object, but when the code goes to fetch the object it comes back nil, so the thumbnail errors.

Code we used to discover this state: ruby id = "cfa08806-5a73-4920-9bcc-2c9df295dd38" r = ChangeSetPersister.default.query_service.find_by(id: id) change_set = ChangeSet.for(r) logical_order = change_set.logical_structure.first members = Wayfinder.for(change_set.resource).members_with_parents logical_order = WithProxyForObject.new(logical_order, members) def add_leaves(a, nodes) nodes.each do |node| if node.nodes.empty? a << node else add_leaves(a, node.nodes) end end end leaves = [] add_leaves(leaves, logical_order.nodes)

Note that the resource didn't have any members in its member ids list that had been deleted, just the proxy nodes.

Possible next steps: - Figure out how these logical nodes persisted after their proxy objects were deleted and prevent that from happening - Change the code to check proxy_for_object instead of proxy.present? - Delete these dead nodes on this resource.
