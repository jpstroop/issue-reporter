I've been trying to isolate the JavaScript packages which might be triggering this. Thus far, I have limited these changes to require only the following:

"@babel/preset-react": "^7.8.3",

James R. Griffin III at 2020-03-12 20:48:16 :

James R. Griffin III at 2020-03-12 21:06:14 :

James R. Griffin III at 2020-03-13 01:04:01 :

James R. Griffin III at 2020-03-13 01:54:51 :

Adding <script src="//cdn.jsdelivr.net/npm/clipboard@2.0.6/dist/clipboard.js"></script> to the app/views/layouts/application.html.erb view partial seems to be the only way I can find to resolve this. Explicitly importing from the source using import ClipboardJS from '../../../node_modules/clipboard/dist/clipboard.js' simply raises the same error (I suspect that this somehow relates to a Webpack configuration setting used to transpile the ES6 from the clipboard dependency).
