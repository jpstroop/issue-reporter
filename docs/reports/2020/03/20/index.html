#3765 Use Pyramidal Tiffs

Creator:
Trey Pendragon
Created:
2020-03-18 20:21:11
Status:
Open (pull request)
Comments:
Trey Pendragon at 2020-03-20 18:40:36 :

The AWS Lambda is shipped and is taking ~ 3 seconds to serve literally as many requests as you can possibly give it. I zoom'd with Michael Klein from Northwestern for a bit and he helped me gain some insight to the stack.

Theirs responds faster. They use the following derivative generation rules:

    const pyramidTiff = await sharp(inputFile, { limitInputPixels: false })
      .resize({
        width: 15000,
        height: 15000,
        fit: "inside",
        withoutEnlargement: true
      })
      .tiff({
        compression: "jpeg",
        quality: 75,
        tile: true,
        tileHeight: 256,
        tileWidth: 256,
        pyramid: true
      })
      .toBuffer();

They limit derivatives to 15k pixels at the largest, jpeg compress to 75 quality, and set tiles to 256x256.

Trey Pendragon at 2020-03-20 20:04:40 :

If I set the JPEG quality to 75 and the tiles to 256x256 the really large images load in a few seconds, but smaller zoomed in tiles take about half a second. The derivatives are much smaller too. Only issue is they take much longer to generate.

#541 move off mysql to postgresql

Creator:
Francis Kayiwa
Created:
2019-05-09 12:15:02
Status:
Open
Comments:
Christina Chortaria at 2020-03-20 19:47:54 :
  1. Dump mysql on production:
    • mysqldump -u user -p password database_name > dump_file.sql
  2. In your local environment:
    1. make sure you are running mysql 5.7
    2. brew install pgloader (https://github.com/dimitri/pgloader)
    3. pgloader mysql://user:pass@localhost/marc_liberation_dev postgresql://user:pass@localhost/marc_liberation_dev
    4. connect to postgres:
      • psql postgres: run: alter database marc_liberation_dev set search_path to marc_liberation_dev, public;
      • export a pg_dump:
        • pg_dump marc_liberation_dev > marc_liberation.dump
  3. on bibdata-staging server
    1. scp the pg_dump from your local
    2. restore the pg_dump:
      • pgsql database_name < marc_liberation.dump
