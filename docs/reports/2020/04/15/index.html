start a mysql 5.7 or earlier as pgloader has issues with later versions brew service stop mysql brew uninstall mysql rm -rf /usr/local/var/mysql rm /usr/local/etc/my.cnf brew install mysql@5.7 brew link --force mysql@5.7 brew services start mysql@5.7 gem uninstall mysql2 gem install mysql2 psql postgres CREATE USER dss; Grant ALL PRIVILEGES on DATABASE dss_staging to dss; ALTER USER dss CREATEDB;

Locally - Change your database.yml dev name to match the server you are targeting dss-staging or dss-production. This saves having to modify the postgres dump to match the new database. mysql -u root <target-server> < dss_production-4-14-2020.sql pgloader mysql://root@localhost/dss_staging postgresql://dss@localhost/dss_staging psql postgres alter database dss_staging set search_path to dss_staging, public; update your database.yml to have the new postgres connection information and check that the dev environment is still working rake server rake dss:solr:deindex rake dss:solr:index Run the test solr server rake server:test RAILS_ENV=test be solr_wrapper -n blacklight-core-test clean #if the above command fails run the tests rspec dump the postgres database pg_dump -U dss dss_staging > dss_staging.dump gzip dss_staging.dump

Update princeton_ansible to point to the postgres database run the playbook

deploy the new postgres code to staging BRANCH=151postgres cap staging deploy

scp the dump file to the server scp dss_staging.dump.gz deploy@dss-staging1:~

Get on the server and create the database as deploy cd /opt/dss-blacklight/current RAILS_ENV=production bundle exec rake db:create RAILS_ENV=production bundle exec rake db:migrate

Stop nginx on server as pulsys sudo service nginx stop

Log into the postgres server psql -U postgresadmin DROP DATABASE dss_staging; Create DATABASE dss_staging;

On the server import the database as deploy cd ~ gunzip dss_staging.dump.gz more app_configs/dss-blacklight psql -U dss -d dss_staging -h lib-postgres3.princeton.edu -f /home/deploy/dss_staging.dump

start nginx as pulsys sudo service nginx start

reindex as deploy /opt/dss-blacklight/current RAILS_ENV=production bundle exec rake dss:solr:deindex RAILS_ENV=production bundle exec rake dss:solr:index
