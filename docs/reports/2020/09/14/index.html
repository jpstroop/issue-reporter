#376 investigate validation of XML files

Creator:
regineheberlein
Created:
2020-09-11 18:05:28
Status:
Open
Comments:
regineheberlein at 2020-09-14 14:56:45 :
  • Nokogiri::XML::RelaxNG has a bug: it interprets <zeroOrMore> as "required" (e.g. throws an error when note is absent but is specified as zeroOrMore). I noticed this can be fixed by changing schema syntax to <optional>.
  • NB: error messages are Nokogiri::XML::SyntaxError objects
  • Doesn't interpret embedded schematron rules
  • runs much slower than in oxygen

oxygen/jing: element "descgrp" incomplete; missing required element "processinfo"

versus

nokogiri: ``` 152:0: ERROR: Expecting an element note, got nothing

152:0: ERROR: Expecting an element processinfo, got nothing

152:0: ERROR: Invalid sequence in interleave

152:0: ERROR: Element descgrp failed to validate content ```

(NB the latter two errors are somewhat redundant, since the missing elements are children of descgrp. Interpreting these error messages may take a little getting used to.)

regineheberlein at 2020-09-14 15:00:17 :

Nokogiri::XML::Schema is an option to validate against an xsd. I'm not sure we'll need it, but it's there.

regineheberlein at 2020-09-14 19:15:16 :

schematron (https://rubygems.org/gems/schematron) running into installation issues: error occurred while installing libxml-ruby (1.1.4), and Bundler cannot continue

If we can get it (or another) to work, we could rewrite the local schema entirely in schematron if we need the interleave rules that Nokogiri::XML::RelaxNG skips.
