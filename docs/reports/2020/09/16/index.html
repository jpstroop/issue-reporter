#4254 Add a date facet to Figgy

Creator:
Kate Lynch
Created:
2020-09-10 12:58:23
Status:
Open
Comments:
hackmastera at 2020-09-16 14:34:09 :

date_created is a string field which according to our tests expects to handle values such as "circa 1865" and "1941-1985".

In order to facet these will need to be normalized into year-only dates. In order to use blacklight-range-limit they will need to be sortable, integer values.

Orangelight uses blacklight-range-limit by pulling the start date from the 008 (logic here) And in Figgy that is the same logic used to populated "date created" (logic here and here).

For finding aid items, figgy pulls the display date into the date_created field but we pull a normalized iso8601 date into the created field. It would be preferable to use this normalized value as the source for the facet.

Implementation proposal:

  1. Create a new Valkyrie field on ScannedResource called pub_date_start_sort (this is what it's called in orangelilght) to use as the facet field. Then pull the value during metadata synchronization.
  2. Or at index time check the source metadata identifier. if it's a bib number copy the date_created field into a new facet field. If it's a component ID extract the year from the created field.
