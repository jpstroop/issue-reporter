#828 Duplicate database entries for custom field override-title_ssim

Creator:
hackmastera
Created:
2020-08-25 12:44:11
Status:
Open
Comments:
Trey Pendragon at 2020-09-24 18:55:09 :

We think the problem is that https://github.com/pulibrary/pomegranate/blob/ac6b8680cdc897cad6c040e53af7243349f15ebc/app/services/iiif_manifest.rb#L144

uses exhibit_custom_fields, which is a hash of all the CustomFields for an exhibit by label. Unfortunately for some exhibits there are somehow two CustomFields with the same label, but different slugs, so the hash's value is overridden. Changing this to use exhibit.custom_fields.pluck(:slug) should fix that and be covered by existing tests.

We don't know how the Spotlight::CustomField with the slug override-title_ssim is created, but if we delete it from existing exhibits the problem shouldn't reoccur.

We re-deleted dups along with those with the slug override-title_ssim
