For this process, I had built the SIPs using the dspace-python repository, with the objective of generating the SIPs in the .tgz (compressed TAR) format. I then tunneled to the server using the following:

ssh -L 1234:[DATASPACE_DEV]:22 pulsys@[BASTION_HOST]

Please note that, in order to support the copying of files over the SSH using scp, I needed to enable the following:

AllowTCPForwarding yes

within /etc/ssh/sshd_config on the server.

Doctoral Dissertations

This approach was similar to what was undertaken for the senior theses submissions, but much more manual and labor-intensive.

In order to retrieve the dissertations, one needed to SSH into the legacy DataSpace production server:

ssh -J [SERVICE_ACCOUNT]@[OIT_BASTION_HOST] [SERVICE_ACCOUNT]@dataspace.princeton.edu
sudo su - root
sudo su - dspace
cd dissertations/etd-ingest/sftp_upload

I then set a batch ID for the new set of dissertations:

export batch_id=20200408

...and create a new directory for importing these:

mkdir $batch_id
cd $batch_id

Unfortunately, at this point, the process is quite manual. I did not have the opportunity to script for this procedure:

sftp  dspace@[DSPACE_SFTP_HOST]

This will prompt one for the password. Please use the password stored in LastPass within Shared-ITIMS-Passwords/dataspace/dataspace.princeton.edu

One will then have the ability to search for the newly-uploaded .zip files:

For the purposes of speed, I was simply manually selecting the files (or, downloading all files and sorting them after they had been copied over the SFTP to the DataSpace server environment).

It should be noted that all files are uploaded by date, and that all files are uploaded in a "batch" style procedure.

I then invoke a Python preprocessing script on the DataSpace legacy server: cd ~/dissertations/etd-ingest python prepare_for_import.py sftp_upload/$batch_id import/$batch_id

This generates a set of directories within dissertations/etd-ingest/import/$batch_id. Once the directories have been generated, they are copied to the dataspace-dev.princeton.edu using scp -r (recursion flag) while tunneling over the SSH (please see above).

My preference has been to transfer the directories on dataspace-dev to /tmp, yielding the following:
