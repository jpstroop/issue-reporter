Notes:

Esmé's script starts off by deleting the repo locally, then cloning it anew. I wasn't really wanting to do this, for fear of losing local work that I'd stashed or accidentally losing a branch I had wip on. So I ran it in a different location.

Steps:

  • Clone Esmé's script to some location, cd there, make it executable
  • Run the script
  • Change the base/target of all open PRs. They won't close automatically but merging them will merge them to master, not to main.
  • Transfer branch protection policies
  • All devs will need to update local settings
    • run git fetch
    • Delete local 'master' branch: git branch -d master
    • Not sure what others will encounter, but as a user of the hub command line tool I had to run this to ensure my PRs will issue to the correct target base: git symbolic-ref refs/remotes/origin/HEAD refs/remotes/origin/main

So the steps would then be:

  1. Run the tool
  2. Create a PR updating the default branch in deploy.rb

When browse lists fail to generate and would like to run the rake task before it is scheduled:

  1. ssh on lib-pulindexer1. Check the crontab and make sure the rake task for the call numbers runs on this box. If not then check lib-pulindexer2 and 3.

  2. While on the right box, run the call number rake task browse:call_numbers. When it finishes count the lines in the file (wc -l orangelightnametitles.sorted) to ensure that there is not a big difference with the previous file.

  3. Run browse:load_call_numbers.
