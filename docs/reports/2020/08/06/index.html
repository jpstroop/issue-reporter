Hi Regine,

Titles are limited to 8704 characters, but you're probably running into this issue, which effectively reduces the maximum length to 255 characters. https://archivesspace.atlassian.net/browse/ANW-757 To be honest, I thought this was fixed. I guess it's time for me to submit a PR.

I'll double-check later today, but I don't think that @unit works in extent statements.

Also, item(s) should work in extents. Could you send me a example where they are failing?

Some other issues:

For extents, the importer looks for "whole" or "part" in @altrender instead of @type. Empty note fields (or <head> only notes) cause the import to fail. Empty subelements in <controlaccess> cause the import to fail. In addition to end dates before beginnings, other invalid @normal dates. Missing <unitdate> at the resource level Missing <unitid> at the resource level Also, I don't know if you've seen Dave Mayo and Kate Bowers' article: https://journal.code4lib.org/articles/12239 Some of the things that they mention have been fixed. (I've been planning on going through and making a list of the current status of everything, but haven't had time.) Even so, it's still a great guide.

Thanks,

Brian
