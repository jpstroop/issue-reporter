#40 move portion/partial info in extent to @altrender

Creator:
regineheberlein
Created:
2020-08-10 20:10:31
Status:
Open
Comments:
regineheberlein at 2020-08-10 21:55:01 :

Upon closer consideration, the portion property of the ASpace container appears to be different from our partiality notation.

"2 boxes (1 partial)" is parsed by the importer as number --> 2 portion --> [whole by default] type --> boxes (1 partial)

I.e. it understands a container type "boxes (1 partial)".

The portion property in ASpace appears to indicate the portion of the resource in the container, not the portion of the container occupied by the resource.

As far as I can see, this means we can't map our partiality references to ASpace, and we can't keep them in the value of extent, either.

#4167 Rename "master" branch to "main"

Creator:
Trey Pendragon
Created:
2020-08-07 18:22:43
Status:
Open
Comments:
hackmastera at 2020-08-10 16:49:15 :

Notes:

Esmé's script starts off by deleting the repo locally, then cloning it anew. I wasn't really wanting to do this, for fear of losing local work that I'd stashed or accidentally losing a branch I had wip on. So I ran it in a different location.

Steps:

  • Clone Esmé's script to some location, cd there, make it executable
  • Run the script
  • Change the base/target of all open PRs. They won't close automatically but merging them will merge them to master, not to main.
  • Transfer branch protection policies
  • All devs will need to update local settings
    • Delete 'master' branch from local clones
    • run git symbolic-ref --short refs/heads/master refs/heads/main to protect yourself when you accidentally try to checkout master (this will give you main instead)
    • Not sure what others will encounter, but as a user of the hub command line tool I had to run this to ensure my PRs will issue to the correct target base: git symbolic-ref refs/remotes/origin/HEAD refs/remotes/origin/main
    • do git branch -r --delete origin/master to remove the local reference to remotes/origin/master
