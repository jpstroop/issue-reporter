#3839 Some Finding Aids fields are not showing up in Figgy and one field is mapped to the wrong place in Figgy

Creator:
Kim Leaman
Created:
2020-04-24 20:29:39
Status:
Open
Comments:
Cliff Wulfman at 2020-05-06 21:19:00 :

Here's where the creator field is coming from:

  • https://github.com/pulibrary/figgy/blob/master/app/services/pul_metadata_services/client.rb#L42 retrieves a record from the findingaids site. NB: this is not the EAD; it is the enhanced EAD that is produced during ingest into pulfa.
  • https://github.com/pulibrary/figgy/blob/master/app/services/pul_metadata_services/pulfa_record.rb#L59 maps creator and publisher to collection_creators, which is an "enhanced" (non-EAD) field in the retrieved record https://github.com/pulibrary/figgy/blob/master/app/services/pul_metadata_services/pulfa_record.rb#L121

This might be the right value to use for a whole collection (though I would suggest the origination element in the EAD's top-level did would be the better value to use), but for constituents, creator should be mapped to the component-level ead:c/ead:did/ead:origination element.
