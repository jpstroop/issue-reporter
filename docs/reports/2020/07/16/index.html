regineheberlein at 2020-07-16 21:40:35 :

From Brian Harrington:

I had been assuming that I would have to write some custom code to load the CPF files, but we have a bigger problem.  Until the agent record improvements in ANW-429 are merged in, the necessary database fields to hold all of the information in your CPF records don't exist in ASpace, and I'm not sure that the extra effort of importing EAC-CPF is worth it if we aren't getting any richer metadata in the authority file.  

Because of this, I think the best bet is to plan on cleaning up the names in the EADs.  Then once the agent record  improvements are in place, you could look at another project to overlay the extra data from the CPF records on top of your existing authorities.
