#822 Calculate catalog_date from Alma records

Creator:
Christina Chortaria
Created:
2020-09-29 13:58:36
Status:
Open
Comments:
Christina Chortaria at 2020-12-03 18:44:17 :

  • [ ] Single Records https://github.com/pulibrary/marc_liberation/issues/748

  1. Get a single alma record with p_avail and e_avail parameters: https://developers.exlibrisgroup.com/console/?url=/wp-content/uploads/alma/openapi/bibs.json#/Catalog/get%2Falmaws%2Fv1%2Fbibs%2F%7Bmms_id%7D

  2. If it has any AVA fields that correspond to p_avail (physical availability), get the list of items to get the <creation_date> . Get the earliest creation_date from the list.

  3. If there are no AVA fields that correspond to p_avail, look for AVE fields that correspond to e_avail. Each AVE field has a portfolio id included. For each AVE field in the API response, you must retrieve the portfolio information. Use the oldest <portfolio><activation_date> from the attached portfolios. See https://developers.exlibrisgroup.com/alma/apis/docs/xsd/rest_portfolio.xsd/

  4. If there is no p_avail or e_avail, use <bib><created_date>.

Christina Chortaria at 2020-12-03 18:44:33 :
  • [ ] Publishing profiles

Bibliographic Enrichment -> Create date subfield 950b Physical Items Enrichment -> Create date subfield 876d Electronic Inventory Enrichment -> Activation date subfield 951w

  1. Similar to the Single Record if there is a 876 field don't use the 951 fields.
  2. If there are no 876 or 951 then use the bib create date: Bibliographic Enrichment 950b.
