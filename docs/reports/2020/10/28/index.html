This adds complexity to the workflow in #64.

The idea of that workflow is to import any bioghist notes that can be robustly associated with name entities into the corresponding Agent records, and delete all bioghist notes from the EAD. This is predicated on the logic that bioghist notes are usually associated with a name (since they make little sense otherwise).

Exempting orphaned biographical notes implies an inverse query that extracts notes that cannot robustly be associated with a name.
