Comments:
Christina Chortaria at 2020-10-29 12:12:22 :

This is related to https://github.com/pulibrary/marc_liberation/issues/874. Since we migrated from mysql to postgres the events page had a random order but not in a specific pattern. We relied on the events_controller index action to list all the events by Event.all. In postgres a select will return all the records but in a random order. We fixed this by forcing an ascending order on the events based on the start attribute of the event. The liberate rake tasks when they pull the json from the events page are checking the start attribute.
