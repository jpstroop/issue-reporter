#2304 Audit Figgy for coin field values being recorded in Figgy but not displayed in the catalog

Creator:
Kevin McElwee
Created:
2020-10-20 15:52:21
Status:
Open
Comments:
Kevin McElwee at 2020-10-26 19:59:54 :

Data and scripts

Catalog JSON, Figgy JSON, and python script (15MB): figgy-catalog-diff.zip

Links

Link to spreadsheet & notes: https://docs.google.com/spreadsheets/d/1mXR8-AcpkN6eeU1A2mTHQKRfcZWE85l93JseYK0ZY9U/edit?usp=sharing

maps figgy -> catalog: https://github.com/pulibrary/figgy/blob/937a33a31dea90f76f1439698748b37d0f0403f1/app/resources/numismatics/coins/orangelight_coin_builder.rb#L37

builds figgy coin decorator: https://github.com/pulibrary/figgy/blob/937a33a31dea90f76f1439698748b37d0f0403f1/app/resources/numismatics/coins/numismatics/coin_decorator.rb

Diff process

  • Gather all unique figgy keys (there are around ~300)
  • Gather all unique catalog keys
  • Strip solr suffixes
    • (Reduces the number of figgy keys to ~90)
  • Find stripped figgy keys that aren't in the catalog
    • (reduces unaccounted figgy keys to ~50)
  • Pair figgy keys with examples, and remove..
    • Keys that store internal info
    • Keys that were renamed when mapped from figgy to catalog
    • Keys that are a combination of other keys
    • Keys who have the same value for all coins (e.g. "claimed_by" is set to "principi" for entire numismatic catalog)
    • (reduces unaccounted figgy keys to ~15)

Possible additions to coin UI

  • Rights information
  • Provenance
  • Arks
  • Loan information
  • Numismatic sub-collection:
    • Unique values: Vermeule, Medals, Firestone, Bell, Antioch, Holzer, Ross Deposit 1, Savage, Museum, Donald, Phillips, Armenian Heritage, Ross, Wu, Near Eastern Studies, Schaaf, Maull, Latin Orient, RBSC
  • Issue information: id & date info
  • Numismatic artist, subject, and note (see below)

Questions:

  • [ ] I assume that everything in the catalog json is displayed on the coin UI, is that a safe assumption? I expect it would take an hour to go through the ~70 keys.
  • [ ] I don't fully understand how figgy works with these. They are numismatic objects: Numismatics::Subject, Numismatics::Note, Numismatics::Citation, and Numismatics::Artist, but I can't find where they'd be mapped to the catalog.
    • For example, Artist ID 7955a4f5-6a12-4127-af52-f359831c626b is mapped to Coin 5232, but there isn't any "Artist" in the UI.
