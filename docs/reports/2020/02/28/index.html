#731 Slavic Collection Rendering Slowly

Creator:
Trey Pendragon

2020-02-10 17:51:02

Open
Comments:
Trey Pendragon at 2020-02-28 08:24:10 :

There were roughly 3400 Spotlight::CustomFields attached to that exhibit for the type field from the JSON-LD, all with the same field but with different slugs. For some reason on import it's not recognizing that those fields already exist, apparently.

For now I just deleted all the dups and it's much faster.

Trey Pendragon at 2020-02-28 08:28:35 :

Selections from Near East Collections has about 6k dups, I'm gonna kill those too.

Trey Pendragon at 2020-02-28 08:31:27 :

Exhibit ID to type dups: [[13, 0], [4, 0], [30, 0], [11, 1], [9, 1], [10, 1], [12, 0], [15, 0], [18, 1], [16, 0], [40, 4], [23, 0], [24, 0], [42, 1], [44, 0], [37, 0], [31, 0], [35, 0], [45, 0], [46, 4], [39, 0], [38, 0], [36, 0], [41, 0], [48, 0], [52, 0], [55, 0], [51, 0]]

So things should be much quicker in general now.

Trey Pendragon at 2020-02-28 09:38:42 :

I'm going to go to sleep. I tracked this down to this:

When an item imports it checks to make sure that a given key doesn't already exist as a label for all the fields that exist. The problem is to do that it uses Spotlight::CustomField#label, which first checks the blacklight config to see if the user has overridden that label.

The labels are mapped here: https://github.com/projectblacklight/spotlight/blob/master/app/models/spotlight/resources/iiif_manifest.rb#L114

The comparison happens here: https://github.com/projectblacklight/spotlight/blob/master/app/models/spotlight/resources/iiif_manifest.rb#L109

The problem is on import the key is @type, but a user has changed it to Type for an existing field. This check should really be making sure the field name doesn't already exist. We should patch that.

IN the meantime, I'm changing those fields for Slavic and Near East Collections back to @type as a label to stop proliferation and hiding them from display.

Trey Pendragon at 2020-02-28 09:41:03 :

Migrating solr data to a new znode.

  1. On each solr box, create a copy of /etc/defaults/solr.in.sh to /etc/defaults/newsolr.in.sh. Update the ZK_HOST param to append the new znode. When it's time to edit this config file, you can just swap in the copy.
  2. Stop solr on all boxes.
  3. On one box only:
    1. Start a tmux session.
    2. cd /opt/solr
    3. bin/solr zk mkroot /solr7 -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    4. bin/solr zk cp -r zk:/clusterstate.json zk:/solr7/clusterstate.json -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181

    6. bin/solr zk cp -r zk:/live_nodes zk:/solr7/live_nodes -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181

    8. bin/solr zk cp -r zk:/overseer_elect zk:/solr7/overseer_elect -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    9. bin/solr zk cp -r zk:/collections zk:/solr7/collections -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    10. bin/solr zk cp -r zk:/configs zk:/solr7/configs -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    11. There are a couple others. Move everything except /solr8 (there is also a /zookeeper but it's only visible form the zookeeper cli).
    12. Check the new znode contents: bin/solr zk ls /solr7 -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    13. Delete those data from their old locations.
    14. bin/solr zk rm -r /clusterstate.json -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    15. bin/solr zk rm -r /aliases.json -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    16. bin/solr zk rm -r /live_nodes -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    17. bin/solr zk rm -r /overseer -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    18. bin/solr zk rm -r /overseer_elect -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    19. bin/solr zk rm -r /collections -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
    20. bin/solr zk rm -r /configs -z lib-zk-staging1:2181,lib-zk-staging2:2181,lib-zk-staging3:2181
